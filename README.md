# Job Automation Pipeline — Data Analyst Edition

Automated job collection, AI-powered scoring, and resume tailoring pipeline for Data Analyst roles.

## How It Works

```
[Job Boards] ──> [Collectors] ──> pending-review.json ──> [Pre-filter] ──> filtered-jobs.json
                                                                                    │
    Indeed ──> jobspy_collector.py                                                   │
    LinkedIn ──> jobspy_collector.py                                          [AI Scoring]
    Google Jobs ──> jobspy_collector.py (Scrapling)                                  │
    Dice ──> dice_collector.py                                                       ▼
                                                                          scored-jobs/YYYY-MM-DD.md
                                                                                    │
                                                                          [Resume Tailoring]
                                                                                    │
                                                                                    ▼
                                                                    tailored-resumes/YYYY-MM-DD/*.docx
```

## One-Time Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Install the anti-detect browser (for Google Jobs scraping)

```bash
python -m camoufox fetch
```

This downloads ~530MB. If you skip it, the collectors still work for Indeed/LinkedIn/Dice but won't fetch Google Jobs or backfill missing job descriptions.

### 3. Configure

```bash
cp config.json.template config.json
```

Edit `config.json`:
- `scoring.target_roles` — job titles to search for (these become search queries)
- `scoring.dealbreakers` — keywords that auto-reject jobs (crypto, clearance, etc.)
- `resume.base_docx_path` — path to your base resume `.docx` file (for resume tailoring)

### 4. Create your resume summary

```bash
cp resume-summary.md.template resume-summary.md
```

Edit `resume-summary.md` with your actual skills, experience, education, and target roles. This is what the AI compares each job against during scoring.

### 5. Customize the scoring rubric

Edit `docs/scoring-prompt.md` and update:
- The "Candidate Profile" section — your skills, achievements, and experience level
- The Location scoring table — cities/regions where you want to work
- The Dealbreakers section — add/remove based on your restrictions

## Daily Workflow

### Step 1: Collect jobs

```bash
python collect_all.py
```

This runs all collectors in sequence (Indeed + LinkedIn + Google Jobs + Dice) and saves results to `pending-review.json`.

**Options:**
```bash
python collect_all.py --role "data analyst"   # single role only
python collect_all.py --hours 24              # last 24 hours only
python collect_all.py --dry-run               # preview without saving
python collect_all.py --no-google             # skip Google Jobs
python collect_all.py --no-dice               # skip Dice
```

Or run the collectors individually:
```bash
python jobspy_collector.py                    # Indeed + LinkedIn + Google Jobs
python dice_collector.py                      # Dice only
```

### Step 2: Score jobs

**With Claude Code (recommended):**

Open Claude Code in this directory and say:
```
score my jobs
```

Claude Code automatically runs the pre-filter, scores all jobs in batches, deduplicates, handles dead/aggregator links, and writes the final report to `scored-jobs/YYYY-MM-DD.md`.

**Without Claude Code (browser-based AI chat):**

First, run the pre-filter manually:
```bash
python score_jobs.py --pre-filter
```

Then upload these 3 files to any AI chat (Claude.ai, ChatGPT, Gemini):
1. `filtered-jobs.json` — generated by the pre-filter above
2. `resume-summary.md` — your resume profile
3. `docs/scoring-prompt.md` — the scoring rubric (in the `docs/` subfolder)

Paste this prompt:

> Score all jobs in filtered-jobs.json against the resume in resume-summary.md using the rubric in scoring-prompt.md.
>
> **Scoring:** Role(0-40) + Skills(0-25) + Level(0-15) + Company(0-10) + Location(0-5) + Salary(0-5) = max 100.
>
> **Rules:**
> - Dealbreakers (see scoring-prompt.md) = score 0, no exceptions.
> - Deduplicate same (title, company) pairs into one entry with all locations listed.
> - Only suggest ATS keywords the candidate actually has — never fabricate skills.
> - Score conservatively when JD text is short or missing.
> - Use `apply_url` if present, otherwise fall back to `url` for the Link column.
> - Flag dead links (⚠ dead link) and aggregator links (⚠ aggregator) in the Why column.
>
> **Output format per job:**
> ```
> JOB: [index] | TITLE: [title] | COMPANY: [company] | LOCATION: [location]
> SCORE: [0-100] | BREAKDOWN: R[role] S[skills] L[level] C[company] Lo[location] Sa[salary]
> CATEGORY: [Data Analysis / BI | Analytics Engineering | Product Analytics | Financial / Quant]
> REASONING: [1-2 sentences] | ATS_TIPS: [keywords for 65+ scores, "N/A" otherwise]
> ```
>
> **Final report:** Group into Tier A (65+) with subcategory tables, Tier B (45-64) summary table, and Skipped (<45) collapsible list. Include 3-5 advisory notes after Tier A.

If the file is too large for the AI chat (100+ jobs), split it:
```bash
python -c "
import json
jobs = json.load(open('filtered-jobs.json', encoding='utf-8'))
for i in range(0, len(jobs), 60):
    batch = jobs[i:i+60]
    name = f'filtered-batch-{i//60 + 1}.json'
    json.dump(batch, open(name, 'w', encoding='utf-8'), indent=2, ensure_ascii=False)
    print(f'{name}: {len(batch)} jobs')
"
```

After scoring, save the report as `scored-jobs/YYYY-MM-DD.md` and clean up:
```bash
python -c "import json, os; json.dump([], open('pending-review.json','w')); [os.remove(f) for f in ['filtered-jobs.json'] + [x for x in os.listdir('.') if x.startswith('filtered-batch-')] if os.path.exists(f)]; print('Cleaned up')"
```

### Step 3: Tailor resumes

**With Claude Code:**
```
tailor my resumes
```

This reads the latest scored report, clusters Tier A jobs by keyword similarity, and generates tailored `.docx` resume variants — one per job cluster. Each variant reframes your existing experience using that cluster's ATS keywords (never fabricates new experience).

Output: `tailored-resumes/YYYY-MM-DD/` with `.docx` files and a `MAPPING.md` showing which resume to use for which job.

## Scoring Rubric

| Category | Weight | What it measures |
|----------|--------|------------------|
| Role | 0-40 | How closely the title matches your target roles |
| Skills | 0-25 | Overlap between JD requirements and your actual skills |
| Level | 0-15 | Whether the seniority level matches your experience |
| Company | 0-10 | Company reputation and data culture |
| Location | 0-5 | Proximity to your preferred locations |
| Salary | 0-5 | Compensation relative to market |

**Tiers:**
- **Tier A (65+)** — Apply ASAP. Use the ATS tips column to tailor your resume.
- **Tier B (45-64)** — Worth applying if Tier A is thin.
- **Skipped (<45)** — Ignore unless something catches your eye.

## File Structure

| File | Purpose |
|------|---------|
| `collect_all.py` | Runs all collectors in one command |

| `CLAUDE.md` | Claude Code workflow instructions |
| `pending-review.json` | Unscored job queue (input) |
| `seen-jobs.json` | Dedup tracking (prevents re-scoring same jobs) |
| `scored-jobs/` | Daily scoring reports (output) |
| `tailored-resumes/` | Generated resume variants per date |

## Customization

### Target Roles
Edit `config.json` → `scoring.target_roles` to change which job titles are searched.

### Dealbreakers
Edit `config.json` → `scoring.dealbreakers` to auto-reject jobs containing specific keywords.

### Scoring Weights
Edit `score_jobs.py` to adjust:
- `ROLE_EXACT` / `ROLE_PARTIAL_KEYWORDS` — which titles score highest
- `SKILLS_KEYWORDS` — which technical skills to look for in JDs
- `TIER1_COMPANIES` — company quality ratings
- `LOCATION_SCORES` — location preferences
- `score_salary()` — salary tier thresholds

### Resume Tailoring
Edit `tailor_resumes.py` to define resume profiles (see comments in file for setup instructions).

## Troubleshooting

- **Unicode errors on Windows:** The scripts force UTF-8 encoding. If you see errors, ensure your terminal supports UTF-8.
- **Google Jobs rate-limited:** Google may block after a few searches. Wait 1-2 hours or use `--no-google`.
- **LinkedIn returns 999:** LinkedIn blocks bot requests. Link validation automatically skips LinkedIn URLs.
- **Indeed links expire:** The pipeline automatically rewrites Indeed links to company careers site search URLs.
- **pending-review.json too large:** Use `--results 15` to reduce jobs per role.
- **filtered-jobs.json too large for AI chat:** Use the split script above to break into batches of 60.
